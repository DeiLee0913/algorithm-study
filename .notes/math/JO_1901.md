# 1901. 소수 구하기 (정올)

- **출처 / 링크:** [정올 1901 - 소수 구하기](https://jungol.co.kr/problem/1901)
- **난이도·태그:** 브론즈1 — 소수, 에라토스테네스의 체
- **상태:** AC
- **메인 아이디어(폴더):** math
- **보조 태그:** sieve, prime, nearest-search
- **풀이 위치:** `math/JO_1901.java`

### 잘한 점

- **에라토스테네스의 체**로 미리 소수 여부를 표시해 두고, 각 `M`마다 O(1)에 판정했다.
- 왼쪽 후보는 **`num - i >= 2`** 로 1을 소수로 잘못 세지 않게 막았다.

### 왜 어려웠는지 / 막힌 지점

- 입력 `M`이 최대 **1,000,000**인데, 합성수일 때 **`M + i` 쪽**으로도 가장 가까운 소수를 찾아야 한다. 체 배열을 **입력 상한과 딱 맞게**만 잡으면 `M = 1,000,000`일 때 `num + 1` 등이 **범위 밖**이거나 판정이 애매해진다. **1,000,100 근처**까지 여유를 두고, 오른쪽은 `num + i < chae.length`로 막는다.
- 처음에는 체 크기를 **1,000,001 수준**으로만 생각했다가, **가장 큰 M에서 오른쪽 탐색**까지 커버하도록 배열을 넓혀야 한다는 걸 깨달았다.
- 거리를 **`for`만으로** 올리려다 **작은 수·탐색 종료 조건**이 복잡해졌다. **`while` + `i++`** 가 이 문제에는 맞다.

### 최종 풀이 요약

- **에라토스테네스의 체**로 충분히 큰 구간까지 합성수를 표시한다. `chae[x] == true`면 합성(또는 0·1), `false`면 소수.
- 각 `M`에 대해:
  - 이미 소수면 `M`만 출력.
  - 아니면 `i = 1`부터 `M - i`, `M + i`가 소수인지 본다. **둘 다** 소수이면 **작은 수부터** 두 개 출력, 한쪽만이면 그 하나만 출력. 못 찾으면 `i++` 후 반복.
- 왼쪽 후보는 **`num - i >= 2`** 일 때만 소수 판정(1은 소수 아님).

### 새로 알게 된 것 / 다음에 쓸 것

- “가장 가까운 소수”처럼 **양쪽으로 퍼지는 탐색**은 거리 변수 + `while`이 읽기 쉽다.
- 입력 최댓값만큼만 체를 깔면 **탐색 방향(+, −)** 에 따라 **한쪽 여유**가 더 필요할 수 있다.
