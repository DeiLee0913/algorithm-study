# 1335. 색종이 만들기(영역구분) (정올)

- **출처 / 링크:** [정올 1335 - 색종이 만들기(영역구분)](https://jungol.co.kr/problem/1335)
- **난이도·태그:** 실버 2 — 분할 정복, 재귀
- **상태:** AC
- **메인 아이디어(폴더):** divide-and-conquer
- **보조 태그:** recursion, quad-split, uniform-check
- **풀이 위치:** `divide-and-conquer/JO_1335.java`

### 잘한 점

- 문제 키워드 **분할 정복**을 보고 접근 방향을 쉽게 잡은 거지만, 재귀에 막연한 두려움이 있었는데, 이번에는 **4분할 재귀 호출 구조**를 거의 완벽하게 짜서 뿌듯했다.

### 왜 어려웠는지 / 막힌 지점

- 처음에는 재귀 인자로 **두 꼭짓점 좌표** `(x1,y1,x2,y2)` 를 썼는데, 중간을 `x2/2`, `y2/2`처럼 잡으면 그건 **전체 종이 기준 절반**일 뿐 **현재 구역의 중간**이 아니라서 잘못된 분할이 된다.

```java
// (틀렸던 호출 예) 현재 사각형 기준이 아닌 전역 절반 좌표
round(x1, y1, x2 / 2, y2);
```

- 인자를 **`(r, c, size)`** 로 두면 네 사분면은 `(r,c)`, `(r, c+size/2)`, `(r+size/2, c)`, `(r+size/2, c+size/2)` 에 각각 `size/2` 를 넘기면 된다.
- 한 칸이라도 색이 다르면 **네 번 재귀한 뒤에는 더 이상 이 구역을 “단색 한 장”으로 세면 안 된다.** 처음에는 재귀 호출 후에 'break'로 잘못 처리했는데, 아래처럼 **즉시 `return`** 해야 자식에서 나눈 결과만 반영되고, 부모가 다시 `white++` / `blue++` 하지 않는다.

```java
divideAndConquer(r, c, newSize);
divideAndConquer(r, c + newSize, newSize);
divideAndConquer(r + newSize, c, newSize);
divideAndConquer(r + newSize, c + newSize, newSize);
return;  // 4분할 후 현재 함수 종료
```

### 최종 풀이 요약

- `N×N` 격자를 읽어 `boards`에 저장한다.
- `divideAndConquer(r, c, size)` : 구역 `(r,c)`~(r+size-1, c+size-1)이 **전부 같은 색**이면 하얀/파랑 카운트 1 증가 후 종료.
- 색이 섞여 있으면 `newSize = size/2` 로 **4분할** 재귀 후 **`return`** (이 구역은 더 이상 단일 색종이로 세지 않음).
- `size == 1`이면 루프 한 번으로 단색 판정된다.

### 새로 알게 된 것 / 다음에 쓸 것

- 분할 정복 격자 문제는 **(시작, 길이)** 가 **(두 꼭짓점)** 보다 사분면 인덱스를 짜기 쉽다.
- 4방 재귀 직후 **`return`** 을 빼먹으면 상위 구역을 또 세는 버그가 난다.

### 참고한 것

- 잘못된 중간 좌표·인자 설계를 **`(r, c, size)`** 로 바꾸는 방향은 **AI** 피드백을 참고했다.
