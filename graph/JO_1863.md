# 1863. 종교 (정올)

- **출처 / 링크:** [정올 1863 — 종교](https://jungol.co.kr/problem/1863) (백준 [1987](https://www.acmicpc.net/problem/1987)과 동일)
- **난이도·태그:** Gold V — 그래프, 연결 요소, BFS, DFS, 유니온 파인드
- **상태:** AC
- **메인 아이디어(폴더):** graph
- **보조 태그:** connected-components, bfs, union-find, rank
- **풀이 위치:** `graph/JO_1863_dfs&bfs.java`, `graph/JO_1863_union&find.java`

### 잘한 점

- “같은 종교 = 같은 집합”을 **연결 요소 개수** 문제로 바로 환원했다.
- **BFS/DFS**와 **유니온 파인드** 두 가지로 모두 풀어 보고, 자바에서 DFS가 위험한 이유를 이해한 뒤 BFS 쪽을 제출용으로 썼다.

### 왜 어려웠는지 / 막힌 지점

- C++에서는 재귀 DFS로 통과하는데, **자바 재귀 DFS는 `n`이 크면 `StackOverflowError`** 가 날 수 있다.
  - 자바는 JVM 위에서 돌고, 스레드당 기본 스택이 대략 **1MB 내외**라 재귀 깊이가 **1만 번 안팎**만 되어도 터지기 쉽다.
  - C++은 OS 스택을 쓰고, 리눅스 채점 서버 기본 스택이 **8MB 정도**인 경우가 많다. 스택 제한을 메모리 제한(이 문제 **256MB**)과 같게 두면 **수만 번** 재귀도 가능한 환경이 있다.
  - 이 문제는 `n ≤ 50,000`이라 **자바에서 재귀 DFS는 비추** → `ArrayDeque`로 **BFS(또는 반복 DFS)** 를 썼다. (`dfs(i)` 호출은 주석 처리)
  - 하지만 dfs 풀이로도 제출해서 정답으로 인정 받았다.
- 유니온 파인드 쪽은 간선마다 `union`만 하고, 마지막에 **`parents[i] == i`인 루트 개수**가 연결 요소 수와 같다. `rank`로 union-by-rank까지 적용했다.

### 최종 풀이 요약

**공통:** 무방향 그래프에서 “같은 종교 쌍” = 간선. **연결 요소의 개수** = 종교 종류의 **최대** 가짓수(각 컴포넌트가 한 종교).

**방법 1 — BFS/DFS (`JO_1863_dfs&bfs.java`)**

- 인접 리스트로 그래프 구성.
- `1 ~ n`을 돌며 `visited[i]`가 false이면 `bfs(i)`(또는 비재귀 탐색) 후 `cnt++`.
- `cnt` 출력.

**방법 2 — 유니온 파인드 (`JO_1863_union&find.java`)**

- 간선 `(a, b)`마다 `union(a, b)` (경로 압축 + rank).
- 전체 처리 후 `parents[i] == i`인 `i`의 개수를 세어 출력.

### 새로 알게 된 것 / 다음에 쓸 것

- **자바 PS:** 정점 수가 큰 그래프(`n` 수만)에서는 재귀 DFS 대신 **BFS·반복 DFS·유니온 파인드**를 먼저 고려한다.
- 연결 요소 개수는 **BFS/DFS** `O(n + m)`, **UF** `O((n + m) · α(n))` (실전에서는 둘 다 `O(n + m)`에 가깝다).
- **α(n)** — 역 아커만 함수. UF에서 `find`/`union` 한 번당 거의 상수 시간이라는 뜻으로 쓰이며, PS 범위의 `n`에서는 사실상 1에 가깝다.
