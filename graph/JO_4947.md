# 4947. 트리의 지름 (정올)

- **출처 / 링크:** [정올 4947 — 트리의 지름](https://jungol.co.kr/problem/4947)
- **난이도·태그:** 골드 3 — 트리, DFS
- **상태:** AC
- **메인 아이디어(폴더):** graph
- **보조 태그:** dfs, tree-diameter, two-pass-dfs
- **풀이 위치:** `graph/JO_4947.java`

### 잘한 점

- 트리 지름을 **DFS 두 번**으로 구하는 정석 알고리즘을 적용했다.
- 양방향 `adjs`와 `prev != next`로 되돌아가기만 막는 DFS 패턴을 그대로 썼다.
- 단말(leaf) 노드를 시작점으로 잡는 시도는 주석으로 남기고, 수학적 보장이 없다는 점을 정리한 뒤 2-pass 방식으로 바꿨다.

### 왜 어려웠는지 / 막힌 지점

- **시작 노드·지름 양 끝을 어떻게 찾을지**가 처음 막혔다. 모든 노드 쌍의 거리를 보면 \(O(N^2)\)이라 \(N \le 100{,}000\)에는 맞지 않는다.
- **잘못된 시도 — 단말 노드에서 시작:** 인접이 1개인 leaf를 찾아 그중 마지막 번호를 `start`로 두고 DFS했다. leaf가 지름의 한쪽 끝일 수는 있지만, **항상 그렇다는 보장은 없다**. 번호가 큰 leaf를 고르는 것도 근거가 없다.
- **AI 힌트로 잡은 핵심 아이디어:** 임의의 노드(보통 `1`)에서 가장 먼 노드 `A`를 찾고, `A`에서 다시 가장 먼 노드까지의 거리가 **트리 지름**이다. 두 번의 DFS로 \(O(V+E)\)에 끝낸다.
- 2차 DFS 전에 `max_distant = 0`으로 초기화해야 한다. 1차에서 쌓인 최댓값이 그대로 남으면 지름을 잘못 출력한다.

### 최종 풀이 요약

1. 입력 간선으로 **양방향** 인접 리스트 `adjs`를 만든다.
2. **1차 DFS:** `dfs(0, 1, 0)` — 임의의 시작점 `1`에서 가장 먼 노드를 찾으며 `max_distant_node`를 갱신한다.
3. **2차 DFS:** `max_distant`를 `0`으로 리셋한 뒤 `dfs(0, max_distant_node, 0)` — 1차에서 찾은 `A`에서 가장 먼 거리를 구한다. 이 값이 지름.
4. `dfs(prev, cur, distant)`는 거리가 커질 때마다 `max_distant`와 `max_distant_node`를 갱신하고, `prev != next`인 이웃만 재귀한다.

### 새로 알게 된 것 / 다음에 쓸 것

- 트리 지름 = **임의 점 → 최원점 A → A에서 최원점**까지 거리. 노드 쌍 전수 조사 없이 DFS 두 번이면 된다.
- leaf에서 시작하는 휴리스틱은 직관적으로 맞아 보여도 **증명된 방법이 아니다**. 지름·최장 경로 문제는 2-pass DFS를 기본으로 기억해 두면 좋다.
- BFS로 거리 배열을 채워도 같은 논리로 2번 탐색하면 되지만, 이 풀이는 `prev`만 넘기는 DFS로 통일했다.

### 참고한 것

- AI 힌트: 트리 지름을 구하는 2-pass DFS 알고리즘.
