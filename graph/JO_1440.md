# 1440. 가장 가까운 공통조상 찾기 (정올)

- **출처 / 링크:** [정올 1440 — 가장 가까운 공통조상 찾기](https://jungol.co.kr/problem/1440) (출처: ICPC Asia Taejon 2002 Regional A)
- **난이도·태그:** Gold IV — 트리, LCA, DFS
- **상태:** AC
- **메인 아이디어(폴더):** graph
- **보조 태그:** tree, parent-array, recursion
- **풀이 위치:** `graph/JO_1440.java`

### 잘한 점

- AI 도움 없이 순수하게 혼자 풀어냈다.
- 루트가 있는 트리를 **부모 배열**(`parents[d] = m`)로 표현하고, 한 노드의 조상 목록을 재귀로 모은 뒤 다른 노드를 위로 올라가며 LCA를 찾는 흐름을 스스로 잡았다.

### 왜 어려웠는지 / 막힌 지점

- **‘공통 조상’에 본인이 포함된다**는 점을 처음엔 명시적으로 생각하지 않았다. 지문에도 *“만약 y가 z의 조상이라면 y가 가장 가까운 공통 조상”* 이라고 되어 있어, **노드 자신도 자신의 조상**으로 취급하는 것이 LCA 문제에서 암묵적으로 통용되는 규칙이다.
  - `findAncestors`에서 `ancestors.add(start)`로 **시작 노드부터** 올라가는 이유가 여기에 해당한다.
- **입력 처리:** 풀이 로직은 맞는데, 직접 숫자를 넣으면 맞고 **예제를 복붙하면 틀리는** 현상이 있었다. AI에게 물어보니 `readLine()` 끝의 공백·줄바꿈 때문에 `parseInt`가 깨질 수 있어서, **`readLine().trim()`** 이 필요했다.

### 최종 풀이 요약

- `N`개 노드, `N-1`줄 간선 → `parents[자식] = 부모` 로 트리 구성 (루트는 `parents[i] == 0`).
- 쿼리 노드 `a`, `b`에 대해:
  1. `findAncestors(a)` — `a`부터 부모를 따라 루트까지 `ancestors` 리스트에 저장 (본인 포함).
  2. `findLCA(b)` — `b`에서 부모 방향으로 올라가며, **현재 노드의 부모**가 `ancestors`에 있으면 그 부모 번호를 LCA로 반환.
- 예제 2 (`a=3`, `b=5`, 답 `3`): 3이 5의 조상이면 3이 가장 가까운 공통 조상.

### 새로 알게 된 것 / 다음에 쓸 것

- LCA·조상 문제에서는 **공통 조상 집합에 두 노드 각각(또는 한쪽 경로의 조상들)이 포함**될 수 있고, “가까운” 조상은 트리에서 위로 올라가며 처음 만나는 공통점으로 보면 된다.
- `parents[]` + 재귀만으로 N ≤ 10,000에서는 충분히 AC 가능. `ancestors.contains()`는 O(깊이)마다 선형이라, 같은 패턴을 반복할 때는 **`HashSet`** 으로 조상 집합을 두면 조회를 O(1)로 줄일 수 있다.
- **방어적 입력 처리:** `BufferedReader`로 한 줄만 읽을 때도 `trim()`을 붙이는 습관. 복붙 입력에 끝 공백·`\r`이 붙으면 로직은 맞아도 런타임/파싱에서 터질 수 있다.
- **후에 해 볼 것:** 깊이·이진 lifting, 또는 **이분 탐색**을 이용한 LCA (오프라인/온라인 고급 기법).

### 참고한 것

- 예제 입력 파싱 문제(`trim`)는 **AI** 피드백으로 해결했다. 알고리즘 풀이 자체는 혼자 풀었다.
