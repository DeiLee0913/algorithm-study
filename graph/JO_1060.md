# 1060. 최소비용신장트리 (정올)

- **출처 / 링크:** [정올 1060 — 최소비용신장트리](https://jungol.co.kr/problem/1060)
- **난이도·태그:** 미정 — 최소 비용 신장 트리(MST), 크루스칼, 유니온 파인드
- **상태:** AC
- **메인 아이디어(폴더):** graph
- **보조 태그:** kruskal, union-find, greedy, sorting
- **풀이 위치:** `graph/JO_1060.java`

### 잘한 점

- 수업에서 배운 **최소 비용 신장 트리(MST)** 흐름을 그대로 잘 옮겼다. 간선을 비용 오름차순으로 정렬한 뒤, 유니온 파인드로 사이클 없이 `N-1`개 간선을 고르는 **크루스칼** 구조가 맞다.
- 입력이 **인접 행렬(대칭)** 이라서, `i < j`인 칸만 간선 리스트에 넣어 **중복 간선**을 막았다.

### 왜 어려웠는지 / 막힌 지점

- 행렬 `mat[i][j]`는 무방향 그래프 비용이라 `(i,j)`와 `(j,i)`가 같다. `i >= j`인 칸까지 전부 넣으면 같은 간선이 두 번 들어가 정렬·루프만 낭비된다.
- 크루스칼에서 트리가 완성되면 간선은 정확히 **`N-1`개**다. 루프 종료 조건을 `taken_edges >= n`처럼 두면 **한 번 더 돌거나** 조건이 애매해질 수 있어, `taken_edges == n - 1`이면 `break`로 맞추는 편이 명확하다.
- 유니온 파인드는 `find`로 루트를 구한 뒤, 두 루트가 다를 때만 합치고 해당 간선 비용을 더한다. 이미 같은 집합이면(`find(from) == find(to)`) 사이클이 생기므로 건너뛴다.

### 최종 풀이 요약

- `N×N` 비용 행렬을 읽으며 `i < j`인 `(i, j, cost)`만 `Edge` 리스트에 담는다.
- 간선을 **비용 오름차순** 정렬한다.
- `parents[1..N]` 유니온 파인드 초기화 후, 간선을 순회하며:
  - 양 끝이 다른 집합이면 `union`하고 비용 합산, 선택 간선 수 증가
  - 같으면 스킵
  - 선택 간선이 `N-1`개가 되면 종료
- 합산한 비용을 출력한다.

### 새로 알게 된 것 / 다음에 쓸 것

- **인접 행렬 MST** 문제는 `i < j`(또는 `i != j` 한쪽만)로 간선을 추출하면 된다.
- 크루스칼 종료는 **`N-1`개 간선 선택**이 트리 완성 조건이다.
- 정점 수가 작을 때(`N ≤ 100`)는 크루스칼 + 유니온 파인드로 충분하다. 밀집 그래프·큰 `N`이면 프림(우선순위 큐)도 비교해 볼 만하다.

### 참고한 것

- 중복 간선 제거(`i < j`)와 루프 종료 조건(`n - 1`) 수정에 **AI** 도움을 받았다.
