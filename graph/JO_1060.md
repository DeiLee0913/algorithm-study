# 1060. 최소비용신장트리 (정올)

- **출처 / 링크:** [정올 1060 — 최소비용신장트리](https://jungol.co.kr/problem/1060)
- **난이도·태그:** Gold IV — 최소 비용 신장 트리(MST), 크루스칼, 유니온 파인드
- **상태:** AC
- **메인 아이디어(폴더):** graph
- **보조 태그:** kruskal, union-find, greedy, sorting
- **풀이 위치:** `graph/JO_1060.java`

### 잘한 점

- 수업에서 배운 **최소 비용 신장 트리(MST)** 흐름을 그대로 잘 옮겼다. 간선을 비용 오름차순으로 정렬한 뒤, 유니온 파인드로 사이클 없이 `N-1`개 간선을 고르는 **크루스칼** 골격은 스스로 잡았다.

### 왜 어려웠는지 / 막힌 지점

- 입력이 **인접 행렬(대칭)** 인데, 처음에는 `i >= j`인 칸까지 전부 간선 리스트에 넣었다. `(i,j)`와 `(j,i)`가 같은 무방향 간선이라 **중복**이 생기고 정렬·루프만 낭비된다. AI 피드백 후 **`i < j`인 칸만** 담도록 고쳤다.
- 크루스칼에서 트리가 완성되면 간선은 **`N-1`개**인데, 종료 조건을 `taken_edges >= n`으로 뒀었다. 한 번 더 돌거나 조건이 애매해질 수 있어, AI 피드백 후 **`taken_edges == n - 1`이면 `break`** 로 바꿨다.

### 최종 풀이 요약

- `N×N` 비용 행렬을 읽으며 `i < j`인 `(i, j, cost)`만 `Edge` 리스트에 담는다.
- 간선을 **비용 오름차순** 정렬한다.
- `parents[1..N]` 유니온 파인드 초기화 후, 간선을 순회하며:
  - 양 끝이 다른 집합이면 `union`하고 비용 합산, 선택 간선 수 증가
  - 같으면 (사이클을 돌게 되므로) 스킵
  - 선택 간선이 `N-1`개가 되면 종료
- 합산한 비용을 출력한다.

### 새로 알게 된 것 / 다음에 쓸 것

- **인접 행렬 MST** 문제는 대칭 행렬에서 **`i < j`일 때만** 간선을 담으면 된다.
  - j < i 쪽은 대칭이라 중복이다.
  - `i == j`는 자기 루프라 트리에 넣을 수 없고, `i < j`만 쓰면 대각선도 자동으로 걸러진다.
- 크루스칼 종료는 **`N-1`개 간선 선택**이 트리 완성 조건이다.
- 정점 수가 작을 때(`N ≤ 100`)는 크루스칼 + 유니온 파인드로 충분하다. 밀집 그래프·큰 `N`이면 프림(우선순위 큐)도 비교해 볼 만하다.

### 참고한 것

- 중복 간선(`i < j`만 담기)과 루프 종료 조건(`n - 1`)은 **AI 피드백**을 받고 수정했다.
