# 18870. 좌표 압축

- **출처 / 링크:** [백준 18870](https://www.acmicpc.net/problem/18870)
- **난이도·태그:** Silver II — 정렬, 좌표 압축, 해시
- **상태:** 학습 정리 (재풀이/복습용)

## 왜 어려웠는지 / 막힌 지점

핵심은 "정렬한 값의 순번"을 빠르게 다시 원본 순서에 매핑하는 것인데, 이 매핑 자료구조 선택에서 막혔다.

- 중복 제거까진 떠올렸지만, 최종 압축 값을 빠르게 찾는 방식이 약했다.
- `List.indexOf()`를 사용하면 매 조회마다 O(N)이라 전체가 느려진다는 점을 놓쳤다.
- 해시맵 경험이 적어서 `값 -> 압축 인덱스` 사전을 바로 떠올리지 못했다.

## 시도했던 것들

- 원본과 정렬본을 분리해서 관리
- 중복 제거 시도
- 압축값 조회를 `indexOf()`로 처리하려고 접근

## 최종 풀이 아이디어 요약

1. 원본 배열을 복사해 정렬한다.
2. 정렬 배열을 순회하며 중복을 건너뛰고, 새로운 값이 나올 때마다 압축 인덱스를 증가시킨다.
3. 이때 `Map<Integer, Integer>`에 `값 -> 압축 인덱스`를 저장한다.
4. 마지막에 원본 배열 순서대로 맵에서 값을 조회해 출력한다.

핵심은 조회를 O(1)에 가깝게 만드는 것이다. `indexOf()`처럼 선형 탐색을 쓰면 입력이 커질수록 급격히 느려진다.

## 새로 알게 된 것 / 다음에 쓸 것

- 좌표 압축은 정렬 자체보다도 **매핑 자료구조 선택**이 성능을 좌우한다.
- "원본 순서 유지 + 정렬 기준으로 번호 부여" 문제는 `Map` 패턴을 먼저 떠올릴 것.
- 중복 제거 이후 원소 위치 탐색을 반복해야 한다면 `indexOf()` 대신 해시 기반 구조를 우선 검토한다.

## 참고한 것

- 해시맵을 이용한 좌표 압축 패턴을 복습하면서 정리했다.
