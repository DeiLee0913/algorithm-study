# 24061. 알고리즘 수업 - 병합 정렬 2

- **출처 / 링크:** [백준 24061](https://www.acmicpc.net/problem/24061)
- **난이도·태그:** Silver II — 정렬, 병합 정렬
- **상태:** 학습 정리 (제출 코드는 draft에 보관)

## 왜 어려웠는지 / 막힌 지점

24060과 같이 병합 정렬을 구현하는데, **24061은 “K번째 저장 직후의 배열 전체”**를 출력한다. 여기서 헷갈린 부분은 `tmp`와 `arr`의 역할이다.

- `merge` 안에서 `tmp`만 채우는 단계가 끝나면, 그 구간 `[left..right]`에 대해 **tmp에는 정렬된 결과가 한 번에 들어 있지만**, `arr`는 아직 바뀌지 않았을 수 있다.
- 문제가 세는 “변경”은 **의사 코드의 `A[i] <- tmp[…]`에 해당하는 대입**마다 1회다. 그래서 `tmp`가 완성된 뒤에도, `arr`로 옮기는 과정은 **한 칸씩** 진행된다.
- K가 작으면 전체 정렬이 끝나기 전에 K번째 대입에서 멈추게 되고, 그때 출력되는 배열은 **앞부분만 새 값이 반영되고 뒤는 이전 값**인 상태가 될 수 있다. 이게 “정렬이 덜 된 것처럼 보이는데 맞는 출력”인 경우가 있다.

## 시도했던 것들

- 병합 정렬 골격은 대부분 구현했다.
- K번째 **상태**를 잡는 부분과, `static`으로 `k`·카운터를 두는지 메서드 인자로 넘길지에서 시간을 썼다.

## 최종 풀이 요약

1. 문제에서 주어진 병합 정렬 의사 코드에 맞춰 `merge`를 구현한다.
2. `A`에 값이 저장될 때마다(문제의 “변경” 횟수) 카운트를 증가시킨다.
3. 카운트가 K가 되는 **그 대입 직후**의 배열 `A`를 통째로 출력한다.
4. 전체 병합이 끝나기 전에 K번을 못 채우면 `-1`을 출력한다.

## 새로 알게 된 것 / 다음에 쓸 것

- “중간 상태” 출력 문제는 **완성된 중간 결과(tmp)**와 **실제로 적용되는 단계(arr 대입)**를 구분해서 생각해야 한다.
- `static`으로 전역 상태를 두면 `k`/`cnt`를 재귀 호출마다 넘기지 않아도 되어 실수가 줄어든다(다만 스레드·재입력은 PS 한 파일에서는 문제 없음).

## 참고한 것

- `tmp`/`arr` 단계와 K가 작을 때의 출력 의미는 LLM 설명과 대화로 정리했다.
