# 1457. 영역 구하기 (정올)

- **출처 / 링크:** [정올 1457 - 영역구하기](https://jungol.co.kr/problem/1457)
- **난이도·태그:** Silver 1 — 그래프 탐색, BFS, 연결 요소
- **상태:** AC
- **메인 아이디어(폴더):** graph
- **보조 태그:** bfs, flood-fill, connected-components, grid
- **풀이 위치:** `graph/JO_1457.java`

## 왜 어려웠는지 / 막힌 지점

- 격자 문제인데 입력 좌표가 \(x,y\)로 주어져서, (0,0) 기준이 좌상단인지/좌하단인지 헷갈렸다.
  - 결론적으로 “빈 칸들의 연결 요소 개수/넓이”만 구하면 되므로, **일관된 방식으로** 좌표를 행/열에 매핑하면 반전 여부 자체는 정답에 영향을 주지 않는다. (인접 관계만 유지되면 됨)
- 각 영역 넓이를 모은 뒤 **오름차순 정렬**해서 출력해야 한다.

## 시도했던 것들

- 처음에는 `size` 같은 int 배열로 영역 번호를 붙이려고 했는데, 이 문제는 영역 넓이만 필요해서 `visited` + BFS 카운트로 단순화했다.
- BFS에서 “방문 여부”와 “직사각형으로 막힌 칸 여부”를 체크 중 하나씩 까먹는 게 있어서 디버깅 하면서 찾아냈다.

## 최종 풀이 요약

1. `m x n` 보드를 만들고(`boards`), 입력으로 주어지는 직사각형 \((x1,y1)~(x2,y2)\) 내부를 **막힌 칸(true)** 으로 표시한다.
  - 코드에서는 각 행 `y`에 대해 `Arrays.fill(boards[y], x1, x2, true)`로 \([x1, x2)\) 구간을 한 번에 채운다.
2. 전체 격자를 순회하면서 아직 방문하지 않았고 막히지 않은 칸을 만나면, 그 칸을 시작으로 BFS를 돌려 **연결된 빈 칸의 개수(넓이)** 를 센다.
  - 시작 칸을 포함해야 하므로 BFS 내부 카운트를 `1`에서 시작한다.
  - 상하좌우 4방향만 연결로 본다.
3. BFS를 시작한 횟수가 “영역 개수”, 각 BFS에서 나온 카운트가 “영역 넓이”이므로 리스트에 모아 정렬한 뒤,
  - 첫 줄에 영역 개수
  - 다음 줄에 오름차순 넓이들을 공백으로 출력한다.

## 새로 알게 된 것 / 다음에 쓸 것
- `Arrays.fill`에서 idx 범위를 지정하려면 (array, idx1, idx2, value) 순서로 인수 입력
