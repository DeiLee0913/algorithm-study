# 1695. 단지번호붙이기 (정올)

- **출처 / 링크:** [정올 1695 — 단지번호붙이기](https://jungol.co.kr/problem/1695) (지문 출처: KOI 전국 1996 초1)
- **난이도·태그:** 그리드 BFS/DFS, 연결 요소, 정렬 (제한 시간 1초·64MB)
- **상태:** AC

## 왜 어려웠는지 / 막힌 지점

- **문제점 1 - 비효율적 카운팅:** void BFS가 탐색하며 몇 번째 group인지 groups[] 배열에 저장해둔 후에,  종료 후 `groups` 이중 루프로 groups 번호 카운트. 탐색을 한 번 더 도는 형태로 구현했다.
  - 구현이 장황해지고 필요 메모리와 시간까지 더 커지는 문제점이 있었다.
- **문제점 2 - 배열 기반 결과 버퍼:** 단지별 수 결과를 `int[]` **고정 크기**로 두어서 최대 단지 수를 예상(25*25 - ... ) 해서 미리 선언해두어야 했고, 정렬 때 **쓰이지 않는 0 패딩**을 걸러야 해서 까다로었다.

## 시도했던 것들

- **개선 1 — BFS 내부 카운팅:** `bfs` 반환형을 `int`로 두고, 큐에 **처음 넣는 시작 칸**과 **새로 방문하는 칸**을 넣을 때마다 `count++`, 탐색 끝나면 그대로 반환. 탐색과 집계를 한 번에 끝낸다.
- **개선 2 — `ArrayList` + 정렬:** 발견한 단지마다 `result.add(bfs(...))`, `result.size()`로 단지 수, `result.sort(Integer::compareTo)`로 오름차순 출력. 실제 단지만 담아 정렬한다.
- **디버깅 포인트 (1695 특이점):**  
  - 한 줄에 `0110...`처럼 붙어 오는 입력 → `charAt(x)` + `Character.getNumericValue` 등으로 인덱스 실수 줄이기.  
  - `points[y][x]==1`이더라도 `groups[y][x]!=-1`이면 이미 속한 단지이므로 **중복 시작 방지**.  
  - 출력: 첫 줄 **총 단지 수**, 이후 줄마다 **단지 크기 오름차순**.

## 최종 풀이 요약

- 격자에서 상하좌우 4방향으로 인접한 `1`끼리 한 연결 요소 → **BFS(또는 DFS) 한 번**에 방문 처리(`groups`)와 **집 개수 반환**.
- 새 컴포넌트는 `groups[i][j] == -1`인 집에서만 시작.
- 크기 목록만 모아 정렬 후 지문 순서대로 출력.

## 새로 알게 된 것 / 다음에 쓸 것

- 연결 요소 문제에서 **카운트는 탐색하며 반환**하면 이차 방문 순회 없이 깔끔하다.
- 개수 불명인 컴포넌트 결과는 `가변 크기 자료형(List등)으`로 모았다가 `sort` 하는 편이 고정 배열·패딩 처리보다 실수가 적다.
