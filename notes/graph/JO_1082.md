# 1082. 화염에서탈출 (정올) — `graph/JO_1802.java`

- **출처 / 링크:** [정올 1082 - 화염에서탈출](https://jungol.co.kr/problem/1082)
- **난이도·태그:** 격자 BFS, 다중 소스, 레이어(분 단위) 시뮬레이션
- **상태:** AC
- **보조 태그:** bfs, multi-source, queue-layer, grid
- **풀이 위치:** `graph/JO_1802.java`

## 왜 어려웠는지 / 막힌 지점

- **1분(한 턴)** 안에서 불과 재우가 동시에 움직이는데, 큐를 한 번에 다 돌려 버리면 “이번 분에 새로 생긴 불/이동”까지 같은 분에 처리해 버려 **시간 압축이 깨진다**. 그래서 **현재 큐 크기만큼만** poll 하며 한 겹(layer)씩 처리해야 한다. (불 큐와 이동 큐 각각 동일 패턴.)
- **경과 시간 `mins`** 를 루프 끝에서 올리면, 탈출 시점에 `+1` 보정이 필요해지기 쉽다. **이동(한 분) 시작 직전에 `mins++`** 하면, `D`에 도달했을 때 그대로 `mins`를 반환하면 된다.
- 시작 칸 `S`는 이미 서 있는 위치인데 **`visited[S] = true` 를 빼먹으면** 방문 판정·중복이 꼬일 수 있다.
- 문제와 예제를 제대로 읽지 않아서 처음에 돌을 빼고 구현했다.

## 시도했던 것들 / 잘한 것들

- 불 확산·재우 이동을 위해 각각 큐를 선언하고 불 먼저 확산 시킨 후에 재우를 이동시키는 구조는 잘 설계했다.
- 처음에는 모든 이동이 끝난 후에 mins 값을 상승시켰는데, 이에 대응되도록 최종 return시 `return mins + 1`**로 잘 보정시켜 반환했다. 하지만 경계 케이스에서 off-by-one 이 날 수 있으므로 mins += 1 -> 불 확산 -> 재우 이동이 가장 좋은 알고리즘이다.

## 최종 풀이 요약

- 불은 `*`, 시작 `S`, 집 `D`, 벽 `X`, 빈 칸 `.` 인 격자에서, **매 분** (1) 불이 상하좌우로 한 칸씩 번지고 (2) 재우도 상하좌우로 한 칸씩 이동한다. 불은 벽·집으로는 안 번짐(문제 조건).
- `fires`, `moves` 두 큐로 BFS. 매 루프를 **1분**으로 본다.
  - 루프 시작에 `mins++`.
  - `fires.size()` 만큼만 poll 하며 인접 `.` 를 `*` 로 바꾸고 불 큐에 넣는다.
  - `moves.size()` 만큼만 poll 하며, 인접 칸이 `D`이면 **`mins` 반환**, `.` 이면 방문 표시 후 이동 큐에 넣는다.
- 이동 큐가 비면 탈출 불가 → `impossible` (`-1`).

## 새로 알게 된 것 / 다음에 쓸 것

- “**동시에** 전파/이동”하는 격자 시뮬레이션은, 큐를 쓸 때 **`size` 스냅샷으로 한 레이어만** 처리하는 패턴이 기본이다.
- 시간을 세는 위치(`mins++`)를 **한 분의 정의와 return 조건**에 맞춰 고정하면 off-by-one 이 줄어든다.
