# 24060. 알고리즘 수업 - 병합 정렬 1

- **출처 / 링크:** [백준 24060](https://www.acmicpc.net/problem/24060)
- **난이도·태그:** Silver III — 정렬, 재귀, 병합 정렬
- **상태:** 학습 정리 (제출 코드는 draft에 보관)

## 왜 어려웠는지 / 막힌 지점

병합 정렬 자체는 짜도, **“몇 번째 저장인지”를 어디서 세느냐**와 **그 순간의 값을 어떻게 읽느냐**에서 계속 꼬였다.

- 문제는 `A[i] = …`로 **원본 배열에 값이 써질 때마다** 1회 저장으로 카운트한다. `merge` 안에서 `tmp`만 채우는 단계와, `tmp`를 다시 `arr`에 복사하는 단계를 구분해야 한다.
- `cnt == k`를 판단하는 순서와 `break` 위치를 잘못 두면, “k번째 저장 직후”가 아니라 **그 다음 반복**에서 멈추는 식으로 흐름이 어긋난다.
- `arr[i++] = tmp[t++]`처럼 **후위 증가**를 쓰면, 한 줄이 끝난 뒤 `i`는 이미 **다음 칸**을 가리킨다. 그 상태에서 `ans = arr[i]`를 하면 **k번째로 저장한 값이 아니라** 다른 칸을 읽는 실수가 난다.
- `merge`가 여러 번 호출될 때마다 `ans`를 `-1`로 되돌리면, 이미 찾은 정답을 **다시 덮어쓸 수 있다**. 초기값만 `-1`로 두고, `cnt == k`일 때만 `ans`를 갱신하는 편이 안전하다.

## 시도했던 것들

- `merge` 구조·`tmp` 배열 분리는 초기부터 방향이 맞았다.
- `arr`로 복사하는 루프에서 `cnt`와 `break` 순서를 조정하며 맞춤.
- 후위 증가 대신 **저장·카운트·정답 갱신**을 한 줄씩 나누는 쪽이 이해와 디버깅에 유리했다.

## 최종 풀이 요약

1. 병합 정렬을 구현한다.
2. `merge`에서 `tmp`에 두 구간을 합친 뒤, **`arr[left..right]`에 다시 쓰는 루프**에서만 저장 횟수를 센다.
3. `arr[i] = tmp[t]` 같은 형태로 한 칸씩 복사할 때마다 `cnt`를 증가시키고, `cnt == k`이면 그때의 `arr[i]` 값을 정답으로 고정한다.
4. 증가 연산자 때문에 인덱스가 밀리지 않게, **저장한 값을 바로 읽을 수 있는 형태**로 작성한다.

## 새로 알게 된 것 / 다음에 쓸 것

- “카운트하는 이벤트”가 정확히 어떤 연산인지(여기서는 `A`에 대한 대입)부터 먼저 고정한다.
- 복잡한 연산자 한 줄에 몰아 넣기보다, **저장 직후**에 `cnt`와 `ans`를 처리하면 실수가 줄어든다.
